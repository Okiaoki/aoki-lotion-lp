# LP模写用 画像差し替えガイド

このREADMEは、`img/slices/pc` / `img/slices/sp` の画像を模写用に差し替えるときの手順です。

## 1. 優先度A（先に差し替える画像）

対象セクション:
- `section-01`
- `section-04`
- `section-05`
- `section-06`
- `section-10`

### PC版とSP版を分ける理由
- 画面比率が異なるため: PCとSPで見せたい範囲が変わる。
- 可読性を守るため: SPは文字・人物が小さくなりやすく、別トリミングが必要。
- CTA導線を最適化するため: FV（`section-01`）や中盤訴求（`section-04`〜`06`）、終盤訴求（`section-10`）は構図差が成果に直結しやすい。
- レイアウト崩れ防止のため: 本LPは1カラム固定なので、画像側で視認性調整する方が安全。

## 2. 推奨サイズとWebP書き出し目安

### 推奨画像幅
- PC: `1000px`（高さは元スライス比率に合わせる）
- SP: `750px`（高さは元スライス比率に合わせる）

### WebP書き出し目安
- 形式: `WebP`（lossy）
- 品質: `q=75〜82`（まずは80目安）
- カラー: `sRGB`
- メタデータ: 可能なら削除（容量削減）

### 1枚あたり容量の目安（実務用の初期値）
- PC: `150〜350KB` 目安
- SP: `80〜220KB` 目安

※ 文字が多い画像は劣化しやすいので、品質を+3〜+5して再書き出し。

## 3. 差し替え順（A → B → C）

### A: 先行差し替え（最優先）
- `section-01/04/05/06/10` のPC/SPを作成・反映
- まず見た目と導線の骨格を確定

### B: 補完差し替え（中優先）
- `section-02/03/07/08/09` を差し替え
- 途中離脱を防ぐため、ストーリーの連続性を整える

### C: 仕上げ差し替え（低優先）
- `section-11/12` を差し替え
- 全体トンマナ、色味、圧縮率の最終調整

## 4. 反映時の命名ルール

ファイル名は既存ルールを維持:
- PC: `./img/slices/pc/section-01.webp` 形式
- SP: `./img/slices/sp/section-01.webp` 形式

番号は `01` 〜 `12` の2桁固定。

## 5. 最低限のチェック項目

- PC/SPで主要テキストが読める
- 人物や商品が端で切れすぎていない
- 画像差し替え後もセクション間の隙間が出ていない
- 初回表示（`section-01`）の表示速度が重くなりすぎていない

## 6. フォーム動作確認手順（ダミー）

1. LPをブラウザで開き、最下部CTA直後の「購入フォーム」までスクロールする
2. 必須項目を空のまま「購入内容を確認する」を押す
3. 各項目下にエラーメッセージが表示され、最初のエラー項目へフォーカスが移動することを確認する
4. メールを不正形式（例: `test@`）で入力し、メール形式エラーが出ることを確認する
5. 郵便番号を不正形式（例: `123-45`）で入力し、7桁エラーが出ることを確認する
6. 正しい値を入力する
   - お名前: 任意の文字列
   - メール: `test@example.com`
   - 郵便番号: `123-4567` または `1234567`
   - 住所: 任意の住所
   - 規約同意: チェックON
7. 再度「購入内容を確認する」を押し、ページ遷移せずフォーム上部に「入力内容を確認してください」が表示されることを確認する
8. DevToolsのConsoleで以下ログを確認する
   - 送信時: `form_submit_attempt`
   - CTAクリック時: `cta_click`（position付き）
   - FAQ展開時: `faq_open`（id付き）

## 販促LPとして追加した要素

- オファー: 上・中・下CTAのオファー条件を `#offerTemplate` + JS注入で共通化（初回価格/通常価格/特典/締切を1か所管理）
- 信頼: フォーム直前に「ご注文前にご確認ください」カードを設置（発送目安/支払い方法/返金条件）。返金条件は `details` で開閉
- 特商法: フッターの特商法・プライバシー・利用規約・会社概要リンクをページ内モーダル表示に変更
- 計測: `cta_click` / `form_view` / `checkout_step_next` / `checkout_step_back` / `legal_open` / `faq_open` を実装
- 運用性: `js/main.js` の `CONFIG` に商品名・価格・送料・返金文言・締切文言を集約し、CTA/フォーム周辺の表示へ反映

## 外部EC誘導型LPの構成

- 本LPは「商品訴求 + 外部決済誘導」の構成です。
- 購入に関わるCTAはすべて外部ECへ遷移します（新規タブ）。
- 外部タブがブロックされた場合は同一タブ遷移にフォールバックします。
- クリック時に「外部の決済ページを開きます」トーストを3秒表示します。

### 差し替えるべき設定値

- `js/main.js` の `CONFIG.checkoutUrl`
- 例: `https://example.com/product/aoki-lotion`

### CTA計測position一覧

- `header`: ヘッダーの「購入へ」
- `top`: FV直後CTA
- `mid`: 中盤CTA
- `bottom`: 最下部CTA
- `floating`: SP追従CTA
- `form`: フォーム内の「外部決済ページへ進む」、確認モーダル内の同ボタン
